# Rule: instruction fidelity

Trigger: an instruction contains a compressed option label ("go with B"), directives that seem to conflict, an ambiguous reuse verb ("rebuild from the existing one"), or an exact literal path — and the next step is costly or hard to reverse.

- **Restate your reading of an ambiguous directive before spending on it.** *(mined 2026-07-12 — 3 redo incidents)* Option labels acquire private definitions ("B" meant one thing to the requester, another to the executor), conflicting directives get silently resolved by guess, and "build on the existing X" splits into *reuse* vs. *reference-then-rewrite*. Each of these cost a full redo. When the interpretation drives costly or irreversible work, restate the reading in one concrete sentence ("I take B to mean …, proceeding on that") and let the requester correct it — during the work, not only at kickoff; the ambiguity often surfaces mid-flight.
- **Literal paths are used literally.** *(mined 2026-07-12 — 3 incidents)* When an instruction names an exact file or directory path, use that path — don't substitute a near-neighbor inferred from naming conventions. If the literal path looks wrong, say so and ask (or state the evidence for the substitution explicitly); silently "correcting" it sent output to a sibling directory and a wrong repo. The same applies when *you* write paths into a dispatch: verify the path exists under the stated working directory before sending it.
